# 우리집톡(가칭) — 앱 UI 디자인 가이드 v1

> 이 문서는 **모든 화면을 통일감 있게 만들기 위한 기준**입니다.
> Claude(나)는 앱 화면을 만들 때 **항상 이 가이드를 지킵니다.**
> 개발업체에도 이 문서를 그대로 넘기면 됩니다.
> 기준 출처: 애플 HIG · 구글 Material 3 · 한국 정부표준 KRDS · WCAG 접근성.

---

## 0. 기본 원칙 (5가지)

1. **깔끔하게** — 그림자·화려한 효과 최소화, 흰 배경 + 얇은 테두리 위주 (한국 앱 표준)
2. **통일** — 같은 요소는 어느 화면에서든 똑같은 색·크기·간격
3. **8의 배수** — 모든 간격·크기는 4, 8, 16, 24… 처럼 8 기준으로 (자로 잰 듯 정돈됨)
4. **손가락 기준** — 누르는 건 최소 48×48 크기 (오타·오작동 방지)
5. **읽기 쉽게** — 글자와 배경 대비 충분히, 정보를 색으로만 구분하지 않기

## 1. 폰트 (Typography)

- **기본 폰트: `Pretendard`** (한국 앱 사실상 표준, 무료·오픈소스, iOS/안드로이드 통일)
  - 폴백: `-apple-system, BlinkMacSystemFont, "Noto Sans KR", sans-serif`
- **굵기는 2가지만**: 일반(400), 굵게(700). (버튼만 예외적으로 600 허용)

| 용도 | 크기 | 굵기 | 줄간격 |
|---|---|---|---|
| 큰 숫자·금액 강조 | 28px | 700 | 1.3 |
| 페이지 제목 (H1) | 22px | 700 | 1.35 |
| 섹션 제목 (H2) | 18px | 700 | 1.4 |
| 본문 (기본) | 16px | 400 | 1.5 |
| 보조 본문 | 15px | 400 | 1.5 |
| 캡션·설명 | 13px | 400 | 1.5 |
| 버튼 글자 | 16px | 600~700 | 1 |
| 탭바 라벨 | 11px | 400 | 1 |

> 규칙: 글자는 **13px 밑으로 내려가지 않기.** 화면마다 같은 용도엔 같은 크기.

## 2. 색상 (Color)

### 브랜드 색 (Primary) — 파랑
> 부동산/금융 앱은 신뢰감 주는 파랑이 표준. **이 색은 나중에 사장님 브랜드 색으로 교체 가능**(변수 한 곳만 바꾸면 전체 반영).

| 이름 | 코드 | 쓰임 |
|---|---|---|
| primary-50 | `#EFF4FF` | 연한 배경, 칩, 정보박스 |
| primary-100 | `#DBE7FF` | 살짝 진한 연배경 |
| **primary-500** | **`#2563EB`** | **기본 브랜드색** (버튼, 강조, 활성 탭) |
| primary-600 | `#1D4ED8` | 눌림 상태, 파란 텍스트 |
| primary-700 | `#1E40AF` | 가장 진한 강조 |

### 회색 (Grayscale)
| 이름 | 코드 | 쓰임 |
|---|---|---|
| gray-0 | `#FFFFFF` | 카드·기본 배경 |
| gray-5 | `#F4F5F6` | 페이지 배경 |
| gray-10 | `#ECEEF1` | 구분선, 카드 테두리 |
| gray-30 | `#B1B8BE` | 입력창 테두리, 연한 테두리 |
| gray-60 | `#58616A` | 강조 테두리 |
| gray-70 | `#464C53` | 보조 텍스트 (설명·메타정보) |
| gray-90 | `#1E2124` | 본문 텍스트 (기본 글자색) |

### 의미 색 (Semantic) — 상태 표시용
| 이름 | 배경 | 텍스트/강조 | 쓰임 |
|---|---|---|---|
| 성공 success | `#ECFDF3` | `#059669` | 완료, 승인 |
| 경고 warning | `#FFFAEB` | `#B45309` | 주의, 확인 필요 |
| 위험 danger | `#FEF3F2` | `#DC2626` | 오류, 삭제, 실패 |
| 정보 info | `#EFF4FF` | `#1D4ED8` | 안내 문구 |

> 규칙: 색 위에 글자를 올릴 땐 **같은 계열의 진한 색**으로 (검정 금지). 대비 4.5:1 이상.

## 3. 간격 (Spacing) — 8의 배수

| 값 | 쓰임 |
|---|---|
| 4px | 아주 붙은 요소 (아이콘-글자) |
| 8px | 작은 간격 (칩 안쪽, 요소 사이) |
| 12px | 카드 안 항목 사이 |
| 16px | **기본 간격** (카드 안쪽 패딩, 화면 좌우 여백, 카드 사이) |
| 24px | 섹션 사이 |
| 32px | 큰 구획 나눔 |

- **화면 좌우 여백: 항상 16px**
- **카드 안쪽 패딩: 16px** (넓게 쓸 땐 20px)

---

## 4. 모서리 둥글기 (Radius)

| 값 | 쓰임 |
|---|---|
| 8px | 뱃지, 작은 칩, 입력창(작게) |
| 12px | **버튼, 입력창** (기본) |
| 14px | 카드 (기본) |
| 999px | 알약 모양 태그, 원형(프로필·검색바) |

> 규칙: **한쪽 면만 테두리 줄 땐 둥글게 하지 않기.**

## 5. 그림자 (Shadow)

- 기본은 **그림자 없이 얇은 테두리**(`1px #ECEEF1`)로 카드 구분 — 한국 앱 표준, 깔끔.
- 떠 있는 느낌이 꼭 필요할 때만 아주 은은하게: `0 2px 8px rgba(0,0,0,0.06)`
- 진한 그림자·네온 금지.
- **예외(2026-07-08): 파란 기본 버튼(.btn-primary)** 은 입체감을 위해 은은한 그라데이션 + 그림자 허용. 모든 버튼(.btn)엔 약한 그림자. → theme.css의 `--primary-grad` · `--btn-shadow` · `--btn-shadow-soft`로 관리(브랜드색 바꾸면 color-mix로 자동 반영).

## 6. 컴포넌트 규격 (Components)

### 버튼
| 종류 | 높이 | 배경 | 글자색 | 테두리 | radius |
|---|---|---|---|---|---|
| 기본 (Primary) | 52px | `#2563EB` | 흰색 | 없음 | 12px |
| 보조 (Secondary) | 52px | `#EFF4FF` | `#1D4ED8` | 없음 | 12px |
| 외곽선 (Outline) | 52px | 투명 | `#1E2124` | 1px `#B1B8BE` | 12px |
| 텍스트 (Ghost) | 44px | 투명 | `#2563EB` | 없음 | — |

- 글자: 16px / 600~700. 좌우 패딩 16px 이상.
- **화면당 "기본(파란) 버튼"은 되도록 1개** — 진짜 중요한 행동에만.
- 하단 고정 CTA 버튼: 화면 맨 아래 고정, 좌우 16px 여백.

### 입력창 (Input / Select)
- 높이 52px · radius 12px · 배경 흰색 · 테두리 `1px #B1B8BE` · 글자 16px
- 안내문(placeholder): `#8A9099` 정도의 연회색, 실제 예시 문구로 ("예: 서울 강남구")
- 선택(focus): 테두리 2px `#2563EB`
- 오류: 테두리 2px `#DC2626` + 아래 빨간 안내문

### 카드
- 배경 흰색 · 테두리 `1px #ECEEF1` · radius 14px · 안쪽 패딩 16px · 카드 사이 12px

### 리스트/매물 카드
- 썸네일 이미지 비율 **4:3** 또는 16:10
- 가격 17px/700 (가장 눈에 띄게) → 주소·평수 13px/`#464C53`
- 태그(역세권 등): 알약/사각 뱃지로

### 뱃지·태그
- 높이 24px · radius 8px(또는 알약 999px) · 글자 12px
- 강조 태그: `#EFF4FF` 배경 + `#1D4ED8` 글자

### 상단바 (Header)
- 높이 56px · 배경 흰색
- 좌: 로고/제목 · 우: 아이콘(알림·메뉴) 최대 3개

### 하단 탭바 (Tab Bar)
- 5개 이하 · 높이 56px + 하단 safe area
- 아이콘 22px + 라벨 11px · 활성 탭은 `#2563EB` + 굵게, 비활성 `#9AA1AC`
- 알림 있으면 아이콘 우상단 빨간 점

### 아이콘 — **MingCute (Fill) 로 확정** (2026-07-08)
- **세트: MingCute (Fill)** · 부드럽고 둥근 솔리드 · 상업용 무료(Apache 2.0)
- 불러오기: Iconify 웹컴포넌트 사용
  - `<script src="https://code.iconify.design/iconify-icon/2.1.0/iconify-icon.min.js"></script>`
  - 사용법: `<iconify-icon icon="mingcute:아이콘이름-fill"></iconify-icon>`
- 크기는 부모 `font-size`, 색은 부모 `color`(currentColor) 상속 → 강조 primary(`#2563EB`)
- **한 세트로만 통일.** 검색도 솔리드(`search-2-fill`). 화살표(`right-line`)만 라인 허용
- 개발 전달용(실제 아이콘 존재 확인 완료) 주요 이름:
  결제=bank-card-fill · 월세/홈=home-4-fill · 관리비/집내놓기=building-2-fill · 세금/내역=bill-fill ·
  환급=wallet-4-fill · 출석=calendar-2-fill · 이사=truck-fill · 수리=tool-fill · 매장/임대인=store-fill ·
  식자재=package-2-fill · 급여=group-3-fill · 경비=file-fill · 한도=chart-bar-fill · 비즈카드=card-pay-fill ·
  혜택=gift-fill · 알림=notification-fill · 메뉴=menu-fill · 내정보=user-3-fill · 등급=leaf-fill ·
  반복=refresh-2-fill · 즉시=flash-fill · 결제요청=link-2-fill · 채팅=chat-3-fill · 기타=grid-fill

## 7. 접근성 (지키면 앱스토어 심사·사용성 유리)

- 누르는 요소 최소 **48×48px**, 서로 8px 이상 띄우기
- 글자-배경 대비 **본문 4.5:1 / 큰글자·아이콘 3:1** 이상
- 정보를 **색만으로** 구분하지 않기 (아이콘·글자 함께)
- 최소 글자 13px

## 8. 화면 공통 패턴

- **Safe area 존중**: 위 노치·아래 홈바 영역 침범 금지 (콘텐츠는 그 안쪽)
- **리스트 → 상세** 흐름: 목록 항목 누르면 상세로, 상단에 ← 뒤로가기
- **빈 화면(Empty state)**: 사과문 말고 안내 — 제목 + 한 줄 설명 + 행동 버튼
   - 예: "아직 저장한 매물이 없어요" + "관심 매물을 담아보세요" + [매물 보러가기]
- **로딩**: 빈 화면 대신 회색 뼈대(스켈레톤) 보여주기
- **하단 CTA**: 중요한 신청/결제 버튼은 화면 하단 고정

## 9. 글쓰기(UX Writing) 톤

- 짧고 명확하게, **버튼은 동사로** ("신청하기", "매물 보기") — "확인" 남발 금지
- 오류는 **무슨 일 + 어떻게** 한 문장 ("이름이 이미 있어요. 다른 이름을 써주세요.")
- 느낌표·과장 자제. 존댓말 유지.

## 11. ⭐ 색·회색·아이콘 (토스 실제 방식) — 정확판

> 출처: 토스 디자인 시스템(TDS). 핵심값 — 파랑 `#3182F6` · 검정 `#191F28` · 연배경 `#F9FAFB`.
> ⚠️ 2026-07-08 재작성: 이전의 "아이콘을 검게/회색으로" 규칙은 **틀렸음(폐기)**. 검은 채운 아이콘은 무겁고 답답함.

### 원칙 0. 회색으로 위계를 "먼저" 잡는다
- 색을 빼고 **회색만으로도** 뭐가 중요한지 보여야 한다. 회색에서 안 되면 색 넣어도 안 된다.
- 색은 마지막에 "전략적 강조"로만.

### 11-1. 강조색은 "파랑 하나"만, 대신 일관되게
- 우리 파랑(`#2563EB` ≈ 토스 `#3182F6`)이 **유일한 상호작용·강조색**.
- 링크·활성 탭·핵심 버튼·기능 아이콘 → 전부 **같은 파랑**으로.
- ✅ 파랑을 "일관되게" 쓰는 건 정돈돼 보임. ❌ 여러 색 섞기가 문제.

### 11-2. 배경은 무채색, "채도 높은 색 블록"을 만들지 말 것
- 배경 = 흰색 / 아주 연한 회색(`#F9FAFB`). 카드 = 흰색 + 얇은 선.
- ❌ **채도 높은 색 타일·블록**을 여기저기 → 이게 진짜 "파랑 범벅"의 원인 (파랑 아이콘이 아니라 **파랑 타일**이 문제였음)
- 아이콘 타일은 **연한 무채색(`#EEF1F4`)** 또는 타일 없이.

### 11-3. 아이콘 색: "파랑" 또는 "연회색" — 절대 검정 금지 ⚠️
- 핵심·기능 아이콘 = **브랜드 파랑**
- 부가·비활성 아이콘 = **연회색 `#B0B8C1`**(밝고 가벼움)
- ❌ **진한 검정(`#191F28`)으로 채운 아이콘 = 무겁고 답답** (지난 실수)
- 토스 탭바: 활성 = 파랑 아이콘 / 비활성 = 연회색 `#B0B8C1` 아이콘 (검정 아님)

### 11-4. 텍스트 위계 = 회색 3단 (⚠️ "글자"에만, 아이콘엔 쓰지 말 것)
| 순위 | 색 | 굵기 |
|---|---|---|
| 1순위 (핵심 숫자·제목) | `#191F28` | 700 |
| 2순위 (설명) | `#4E5968` | 400 |
| 3순위 (부가·힌트) | `#8B95A1` | 400 |
- 대비를 크게 벌릴 것. 이 진한 검정은 **글자 전용** — 아이콘엔 금지.

### 11-5. 반경(radius) — 토스 기준
- 입력 8 / 카드 12 / 시트·바텀시트 16

### 11-6. 우리 앱 적용 규칙 (요약)
- 아이콘: 파랑(핵심) 또는 연회색 `#B0B8C1`(부가). **검정 금지.**
- 타일: 연무채색 `#EEF1F4` 또는 없음. 채도 높은 색 블록 최소화.
- 버튼: 핵심 CTA만 파란 채움(화면당 1개), 유틸은 회색 알약.
- 위계: 회색+굵기+여백으로 먼저 잡고, 파랑은 강조로만.
- 토큰: 아이콘 `--primary`/`--icon-off`, 타일 `--tile`, 글자 `--ink`/`--sub`/`--placeholder`.
