# 우리집톡(가칭) — UX·가독성·사용성 가이드 v1

> 앞의 [DESIGN_GUIDE.md](DESIGN_GUIDE.md)가 **"무슨 색·크기를 쓸지"(재료)** 라면,
> 이 문서는 **"어떻게 배치해야 읽기 쉽고 쓰기 편한지"(원칙)** 입니다.
> Claude(나)는 화면을 만들 때 이 원칙을 함께 지킵니다. 개발업체에도 넘기면 됩니다.
> 기준: 닐슨 사용성 10원칙 · 애플/구글 UX · 모바일 가독성 연구.

---

## 0. 가장 중요한 원칙 5개 (다 잊어도 이건 기억)

1. **직관성** — 설명 없이도 뭘 누르면 되는지 보이게
2. **일관성** — 같은 건 어디서든 똑같이 (버튼·색·용어)
3. **가시성** — 지금 상태·다음 행동이 항상 보이게
4. **즉각 피드백** — 누르면 반드시 반응 (색 변화·로딩·완료 알림)
5. **접근성** — 누구나 쉽게 (큰 터치영역·충분한 대비·쉬운 말)

---

## 1. 시각적 위계 — "뭘 먼저 보게 할까"

사람 눈은 위→아래, 큰 것→작은 것 순으로 봅니다. 중요한 걸 눈에 띄게.

- **중요한 것 = 크게·굵게·위에·진하게.** 덜 중요한 건 작고 연하게.
- **한 화면에 진짜 강조는 1~2개만.** 다 강조하면 아무것도 안 보임.
   - 예: 매물카드에서 **가격**이 제일 크게, 주소·평수는 작게.
- **관련 있는 건 묶기(그룹핑).** 묶을 땐 **여백**으로 (선보다 여백이 깔끔).
- **파란 기본 버튼은 화면당 1개** — "이걸 누르세요" 신호.

---

## 2. 가독성 — "글이 술술 읽히게"

- **줄간격 넉넉히 (1.5).** 글자가 붙으면 답답.
- **한글 본문은 왼쪽 정렬.** 가운데 정렬은 짧은 제목·버튼에만.
- **한 줄이 너무 길지 않게.** 좌우 16px 여백 유지.
- **회색 글자를 너무 연하게 쓰지 않기.** (배경과 대비 4.5:1 이상)
- **문단은 짧게, 핵심만.** 긴 설명은 접어두고 "더보기".
- **전부 대문자·전부 굵게 금지.** 강조가 사라짐.

---

## 3. 엄지 영역(Thumb Zone) — "한 손으로 편하게"

폰은 대부분 한 손·엄지로 씁니다. 화면 아래쪽이 엄지가 편한 구역.

- **자주 누르는 버튼은 화면 아래쪽에.** (하단 탭바, 하단 고정 CTA)
- 화면 위쪽 구석은 엄지가 안 닿음 → 자주 쓰는 기능 두지 않기.
- **위험한 행동(삭제·탈퇴)은 아래 구석 or 확인 한 번 더** — 실수 방지.

---

## 4. 사용성 10원칙 (닐슨) — 쉬운 말로

| # | 원칙 | 우리 앱에선 |
|---|---|---|
| 1 | 지금 상태 보여주기 | 로딩 중·저장됨·몇 단계 중 몇 번째인지 표시 |
| 2 | 익숙한 말 쓰기 | 전문용어 X ("전용면적" 옆에 쉬운 설명) |
| 3 | 빠져나갈 길 | 항상 ← 뒤로·취소 버튼 |
| 4 | 일관성 | 같은 버튼은 늘 같은 자리·같은 모양 |
| 5 | 실수 예방 | 삭제 전 "정말요?" 확인, 되돌리기 |
| 6 | 기억시키지 말기 | 외우게 하지 말고 **선택지를 보여주기** (드롭다운·최근검색) |
| 7 | 지름길 | 자주 쓰는 기능은 홈에서 바로 |
| 8 | 군더더기 빼기 | 안 쓰는 정보·버튼 과감히 제거 |
| 9 | 오류는 사람 말로 | "오류 500" X → "잠시 후 다시 시도해주세요" + 해결법 |
| 10 | 도움말 | 헷갈리는 곳엔 작은 안내 문구·물음표 |

---

## 5. 입력·폼(Form) — "묻는 건 최대한 적게, 쉽게"

- **한 화면에 한 가지만 (One thing per page).** 한꺼번에 다 묻지 말고 단계로 쪼개기.
- **입력칸은 세로 한 줄로.** 좌우로 나란히 두지 않기 (모바일에서 헷갈림).
- **입력하는 즉시 검사(inline).** 다 쓰고 "제출" 눌러서야 빨간불 X → 칸 벗어나면 바로 알려주기.
- **꼭 필요한 것만 물어보기.** 필드 하나 늘 때마다 이탈률 증가.
- **라벨은 칸 위에**, 안내(placeholder)는 **진짜 예시**로 ("예: 서울 강남구 역삼동").
- **키보드 맞추기.** 전화번호·금액 칸은 숫자 키패드가 뜨게.

---

## 6. 빈 화면(Empty State) — "비었을 때가 기회"

목록이 비었을 때 **하얀 화면만 두지 않기.** 3가지에 답하기:

1. **이게 뭔가요?** (여긴 저장한 매물이 모이는 곳)
2. **왜 중요한가요?** (관심 매물을 모아두면 비교가 쉬워요)
3. **뭘 하면 되나요?** → **버튼** (매물 보러가기)

> 사과문("아직 없어요…") 말고 **초대**로. 제목 + 한 줄 설명 + 행동 버튼.

---

## 7. 피드백 — "누르면 반드시 반응"

- 버튼 누르면 **색 변화/눌린 느낌** 즉시.
- 시간 걸리는 작업은 **로딩 표시** (빈 화면 X → 회색 뼈대 스켈레톤).
- 끝나면 **결과 알림** (완료 토스트 / 실패 안내).
- 되돌릴 수 있는 행동은 "실행취소" 잠깐 띄우기.

---

## 8. 인지부하 줄이기 — "생각하게 만들지 않기"

- **선택지를 줄이기.** 많으면 못 고름 → 추천·인기 먼저.
- **단계적 공개(progressive disclosure).** 처음엔 핵심만, 필요하면 "더보기".
- **익숙한 패턴 쓰기.** 남들 다 쓰는 위치(장바구니 우상단, 탭바 아래)를 따르기.
- **한 번에 하나씩.** 화면마다 목표 하나.

---

## 9. 부동산 앱 특화 팁

- **매물 목록은 사진 → 가격 → 위치 순으로 스캔되게.** 가격이 가장 큼.
- **필터를 쉽게** (지역·가격·방 개수) — 자주 쓰는 필터는 상단 고정.
- **신뢰 요소를 보이게** — 실거주 후기, 인증 뱃지, 최근 실거래가.
- **지도 ↔ 목록 전환**을 한 버튼으로 쉽게.
- 돈·계약 관련 화면은 **특히 명확하게** (금액·조건 크게, 애매한 말 금지).

---

## ✅ 화면 만들 때 체크리스트 (매번 확인)

- [ ] 이 화면의 **목표 하나**가 분명한가?
- [ ] 제일 중요한 게 **제일 눈에 띄는가?**
- [ ] **파란 버튼은 1개**인가? (그게 하라는 행동인가)
- [ ] 누르는 곳이 **48px 이상**, 서로 안 붙었나?
- [ ] 글자-배경 **대비 충분**한가? 연회색 남발 안 했나?
- [ ] **뒤로/취소** 길이 있나?
- [ ] 비었을 때·로딩·오류 화면을 준비했나?
- [ ] 물어보는 입력, **더 줄일 수 있나?**
